About 4Bytes Framework

4Bytes is designed to change the way you make PHP applications. This easy-to-use framework is designed around being the fastest, easiest, and sexiest framework. Yes, I said sexy when describing a PHP framework.

The main goal of 4Bytes is ease-of-use. Nothing is more frustrating, especially to a new PHP developer, than trying to learn a complex framework on top of learning PHP. With the 4Bytes Framework, you get a really powerful system, whilst being easy to read/code.

4Bytes is a module driven framework. You can make modules, or download pre-made ones. The default module in every installation is the index module. The index module handles the default page requests.

Check out the code snippets below to get a taste of 4Bytes:

Here is your basic 'Hello, World!' program created by the index module.
User modules are located in: usr/modules/custom/

  1. class Index extends Module{
  2. function __construct(){
  3. echo Hello, World!";
  4. }
  5. }

4Bytes Framework has 3 main features to learn:

  • Modules - Where you write your code
  • Views - Where you write your HTML
  • Layouts - Where you combine your views

Modules

Modules are what make 4Bytes so extensible. A module is simply a PHP class that is registered as a module within the system. By registering your module with 4Bytes, it is able to instantiate it, and call methods within it.

Modules are used to create menu_items. A menu_item is a piece of code that tells the system what URLs are mapped to which function.

An example menu_item.

  1. $menu_item[] = array(
  2. 'path' => 'blog',
  3. 'module' => 'index',
  4. 'function' => 'blog',
  5. )

This menu_item allows the URL http://mydomain.com/blog to run the Index::blog(); function. How cool is that?

Example of blog callback:

  1. class Index extends Module{
  2. public function blog(){
  3. echo 'Welcome to the Blog page';
  4. }
  5. )

The menu_item system makes it really easy to do AJAX callbacks. You can easily return JSON results to build out awesome APIs.

Example of JSON callback:

  1. //in settings.php
  2. $menu_item[] = array(
  3. 'path' => 'update/myapi.json',
  4. 'module' => 'index',
  5. 'function' => 'update_myapi',
  6. )
  7. //in index.module.php
  8. class Index extends Module{
  9. public function update_myapi(){
  10. global $sys; //main system object in 4Bytes
  11. $sys->modules->page->set_headers(array(
  12. 'Content-type'=>'application/json',
  13. ))->send_headers();
  14. $json = array('status'=>'okay','message'=>'it works!');
  15. echo json_encode($json);
  16. }
  17. )
